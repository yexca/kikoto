# Architecture Decision Records

ADRs record durable design decisions.

- [ADR-0001: Unified work model](ADR-0001-unified-work-model.md)
- [ADR-0002: Source presence boundary](ADR-0002-source-presence-boundary.md)
- [ADR-0003: SQLite first](ADR-0003-sqlite-first.md)

## When To Add An ADR

Add an ADR when a decision changes long-term architecture, persistence,
deployment, source boundaries, or developer workflow. Routine implementation
details should stay in code comments or area docs.
