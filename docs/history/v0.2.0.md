# v0.2.0

Kikoto v0.2.0 expands the personal audio library with a typed workflow
composer, richer execution visibility, personalized discovery, and more
consistent playback and appearance controls.

## Workflows And Activity

- Added an editable typed DAG composer with validated ports, cycle prevention,
  persisted node positions, undo/redo, reusable workflow composition, and
  filesystem, source-poll, and calendar triggers.
- Added immutable execution graph snapshots and node-level Activity progress,
  values, logs, errors, and retry visibility.
- Kept Legacy workflow definitions readable as reconstructed linear canvases;
  migration remains intentionally deferred and Legacy definitions stay
  read-only.
- Improved Activity empty states and progressive list sizing so empty status
  categories no longer flash as tall loading regions.

## Library And Playback

- Added personalized recommendation sorting and explainable recommendation
  markers based on listening state, tags, voices, circles, and favorites.
- Added Recently played browsing, shared age-rating badges, and refined work
  detail source actions and metadata layout.
- Added Hidden, Preview, and Full lyrics modes and improved player state
  persistence and interaction feedback.

## Appearance And Interaction

- Added pink, blue, and green accent themes alongside Light, Dark, and System
  appearance modes; pink remains the default.
- Added consistent hover and pressed feedback to player controls and shared
  buttons.

## Release And Licensing

- Unified frontend, backend, and Android semantic versions through the root
  `VERSION` file.
- Added a same-commit CI gate before tag publication and delayed GitHub Release
  creation until image and APK publication succeed.
- Licensed Kikoto under AGPL-3.0-only. ComfyUI is acknowledged as a workflow
  canvas design reference; Kikoto does not include or adapt ComfyUI source
  code.
