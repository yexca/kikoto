Kikoto v0.2.1 improves work-detail browsing, responsive playback, update
guidance, and media-indexing reliability.

## Library And Work Detail

- Unified local and remote work-detail behavior while preserving
  language-edition families during metadata search and presentation.
- Refined source switching, file presentation, work metrics, directory
  breadcrumbs, and responsive loading structure.
- Kept the desktop sidebar fixed while allowing compact-player pages to expose
  their final content without overlapping controls.

## Playback And Mobile

- Improved compact and full-player layouts, long-title marquees, mobile safe
  areas, and playback workflows across desktop, Android, and iOS.
- Added clearer Android update notices with direct GitHub Release guidance.
- Coalesced playback progress updates and retained loaded work details when a
  transient database-busy response occurs.

## Operations And Reliability

- Reduced duplicate work-detail queries and SQLite write contention during
  local media indexing.
- Added index-task deduplication, bounded heavy writes, fewer per-file SQL
  statements, and timing logs for slow indexing operations.
- Expanded concurrent indexing and responsive player regression coverage.
- Allowed a release tag to be pushed with its `main` commit by waiting for the
  same-commit CI run before publishing images and APKs.
