# History

This directory is reserved for public historical notes and release-oriented
summaries.

Durable design decisions should go in [ADRs](../decisions/index.md). Current
behavior should be documented in product, architecture, operations, or
development docs.

## What Belongs Here

- Public release summaries.
- Migration notes for users.
- Retrospectives that are useful after the implementation detail has moved on.

Avoid using history pages as the only source of truth for current behavior.
