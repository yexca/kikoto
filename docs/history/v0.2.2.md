Kikoto v0.2.2 makes Library search more predictable, adds normalized
commercial metadata and a read-only demo runtime, and reduces database pressure
during playback.

## Library And Work Detail

- Reworked server-side text search around normalized titles, work codes,
  language-edition aliases, circles, tags, and voice credits. Ordinary text and
  code searches no longer scan raw metadata snapshots while serving a Library
  request.
- Persisted provider-declared edition codes so manual code searches can resolve
  related language editions without creating duplicate works.
- Added dedicated rating and sales fields, plus normalized regular and current
  prices. Work cards show the current price and identify permanently free
  works.

## Sources And Demo Mode

- Added DLsite regular and current price collection, and mapped the current
  price exposed by compatible Remote Sources without creating price history.
- Unified runtime selection under `KIKOTO_MODE`, with `production` as the
  default and explicit `development` and `demo` modes.
- Limited Demo mode to all-ages, permanently free works while allowing admitted
  full media playback. Compatible Remote Sources apply the same restriction
  through their filtered search contract.
- Made Demo sessions read-only across the HTTP API and disabled settings,
  workflow, schedule, run, and review mutations in the interface.

## Metadata And Reliability

- Recorded products that are unavailable from DLsite and excluded them from
  later bulk metadata refreshes instead of repeatedly creating review work.
- Cached resolved media stream targets briefly to reduce repeated SQLite reads
  when switching or replaying tracks.
- Revalidated restored player queues against current media locations after a
  page reload.

## Playback

- Reduced server progress checkpoints to a fixed 30-second interval independent
  of playback speed, while still flushing important seek, pause, track-change,
  completion, and page-visibility events immediately.
- Added frequent browser-local progress checkpoints and restored the newer of
  local and server progress after reloads.
- Collapsed each server progress update into a single SQLite upsert, further
  reducing read and write contention during continuous playback.
